## 모델링

- 키오스크를 통한 메뉴 주문을 구현한다.
- 연관 메뉴
    - [ ] 연관 메뉴는 이름으로 이뤄져 있다.
    - [ ] 연관 메뉴는 등록이 가능하다.
    - 불변식
        - [ ] 연관 메뉴의 이름은 공백이면 안된다.
- 메뉴
    - [ ] 메뉴는 이름, 가격, 표출 여부, 연관 메뉴, 여러 메뉴 구성으로 이뤄져 있다.
    - [ ] 메뉴는 등록, 가격 수정, 미표출, 표출이 가능하다.
    - 불변식
        - [ ] 메뉴의 이름은 공백이여서는 안되고, 비속어가 포함되면 안된다.
        - [ ] 메뉴의 가격은 음수일 수 없다.
        - [ ] 메뉴는 연관 메뉴에 필수적으로 등록 되어야 한다.
        - [ ] 메뉴의 모든 메뉴 구성의 상품 수량은 1개 이상이어야 한다.
        - [ ] 메뉴의 가격은 모든 메뉴 구성의 합이다.
        - [ ] 메뉴를 표출 할 때 메뉴의 가격은 1원 이상이어야 한다.
- 상품
    - [ ] 상품은 이름과 가격이 있다.
    - [ ] 상품은 등록, 가격 수정이 가능하다.
    - 불변식
        - [ ] 상품의 이름은 공백이여서는 안되고, 비속어가 포함되면 안된다.
        - [ ] 상품의 가격은 음수일 수 없다.
    - 가격 수정
        - [ ] 메뉴의 총 가격을 다르게 한다면 메뉴는 미표출 한다.
- 메뉴 구성
    - [ ] 메뉴 구성은 상품과 상품 수량이 있다.
- 주문
    - [ ] 주문은 수령 방법, 상태, 여러개의 주문 목록이 있다.
    - [ ] 주문은 주문 요청, 승인, 제공, 배달 시작, 배달 완료, 주문 종료됨 이 가능하다.
    - 불변식
        - [ ] 주문은 수령 방법이 필수이다.
        - [ ] 주문은 주문 목록은 필수이다.
        - [ ] 주문 수령 방법이 배달일 경우 배달지 주소는 필수이다.
        - [ ] 주문 수령 방법이 매장내 식사의 경우 테이블은 필수이다.
    - 주문 요청
        - [ ] 매장 식사의 경우가 아닐 경우 주문 목록의 메뉴 수랑은 필수 이다.
        - [ ] 표출된 메뉴만 가능하다.
        - [ ] 주문 당시의 가격과 메뉴 가격은 같아야 한다.
        - [ ] 일시를 기록한다.
        - [ ] 상태는 주문 요청 됨 이다.
    - 주문 승인
        - [ ] 상태는 주문 요청 됨 이어야 한다.
        - [ ] 주문 수령 방법이 배송일 경우 라이더 배정이 필수이다.
        - [ ] 상태는 승인됨 이다.
    - 주문 제공
        - [ ] 상태는 승인됨 이어야 한다.
        - [ ] 상태는 제공됨 이다.
    - 배송 시작
        - [ ] 주문 수령 방법이 배송 이어야 한다.
        - [ ] 상태는 제공됨 이어야 한다.
        - [ ] 상태는 배송중 이다.
    - 배달 완료
        - [ ] 주문 수령 방법이 배송 이어야 한다.
        - [ ] 상태는 배송중 이어야 한다.
        - [ ] 상태는 배송됨 이다.
    - 주문 종료
        - [ ] 주문 수령 방법이 배송의 경우 상태는 배송됨 이어야 한다.
        - [ ] 테이크 아웃 또는 매장내 식사의 경우 상태는 제공됨 상태이어야 한다.
        - [ ] 매장내 식사의 경우 테이블을 청소한다.
        - [ ] 상태는 주문 종료됨 이다.
- 주문 목록
    - [ ] 주문 목록은 메뉴와 메뉴 수량을 가지고 있다.
    - [ ] 주문 목록은 메뉴와 메뉴 수량으로 총 가격을 계산한다.
- 테이블
    - [ ] 테이블은 이름과 인원수를 가지고 있다.
    - [ ] 테이블은 인원수와 착석 여부를 가지고 있다.
